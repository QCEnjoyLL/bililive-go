# v1.1.15 更新说明

本版本重点修复 BoyFriend / doppiocdn HLS 录制中“日志显示下载失败为 0，但成片仍然跳秒、卡顿”的问题。建议正在使用 `v1.1.14` 测试 HLS 录制的用户升级。

## HLS 录制稳定性修复

- 修复 MOUFLON 分段 hash 解析过窄的问题。部分真实分段 hash 会包含 `+`、`-`、`_` 等 base64 风格字符，旧逻辑只识别字母数字，导致这些分段被静默忽略。
- 修复“分段已在 playlist 中出现，但下载稍慢时被过早跳过”的问题。现在未知缺口会短时间等待以追直播，已发现并正在下载的分段会给更长等待时间，避免 1-4 秒级网络抖动造成误丢段。
- 优化 HLS playlist 轮询和分段调度诊断，继续输出覆盖率、确认丢段、疑似漏看、下载队列、写入队列、写入阻塞、最大下载耗时等关键指标。
- 自愈引导逻辑同步支持包含特殊字符的分段 hash，避免后续 MOUFLON 密钥变化时配对失败。

## 卡顿诊断结论更清晰

- 当 `确认丢段` 和 `疑似漏看` 均为 0，但视频仍出现“画面短暂停住、音频继续”的情况，通常说明源流视频轨本身短时间没有新帧，而不是 HLS 分段漏录。
- 本版本附带媒体缺口分析脚本 `scripts/analyze-media-gaps.ps1`，可用 ffprobe 检测录制文件中的音视频时间戳缺口，辅助区分“录制漏段”和“源流视频断帧”。

## WebUI 细节优化

- 录制环境可用后，顶部辅助组件提示会进入倒计时并自动隐藏，避免长期占用页面空间。
- 收窄顶部辅助提示条，优化顶部栏显示观感。
- 在 `Bililive-go` 标题前增加图标标识。

## 本地验证结果

- `go test ./src/pkg/parser/hlsmouflon` 通过。
- 使用 `jacques_cardiet` 连续录制多轮测试，HLS 诊断日志显示 `确认丢段 0/累计 0`、`疑似漏看 0`、覆盖率 100%。
- 成片检测中音频缺口为 0；剩余少量 video-only 缺口经 ffprobe 确认是源流视频帧本身断档，音频连续，非 HLS 分段漏录。
