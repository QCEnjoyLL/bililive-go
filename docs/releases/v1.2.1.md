# v1.2.1 更新说明

本版本是一次 WebUI 设置页体验修复，重点解决长设置页面保存不方便、保存后位置丢失的问题。

## 设置页保存体验

- 设置页面的保存按钮现在会固定显示在页面底部，不需要每次修改配置后再滚动到页面最底部才能保存。
- 全局设置、通知设置和 YAML 模式都使用同一套浮动保存按钮样式，视觉上与当前主题保持一致。
- 移动端页面也保留底部浮动保存按钮，并预留底部空间，避免按钮遮挡最后几项配置。

## 保存后位置保持

- 修复设置页保存后自动跳回顶部的问题。
- 保存配置后会静默刷新配置数据，不再把整个设置页临时替换成加载状态，从而避免页面内容卸载导致滚动位置重置。
- 滚动位置会同时记录并恢复主内容区、浏览器窗口和文档滚动容器，适配不同浏览器下的实际滚动行为。
- 顶部“刷新”按钮也会尽量保留当前滚动位置，刷新状态时不打断正在查看的配置区域。

## 开发验证与缺陷修复

- 修正监听器单元测试的旧预期：当前下播判定包含“连续两次离线才确认下播”的防抖逻辑，测试现在会覆盖第一次离线保持在线、第二次离线触发下播事件的行为。
- 发布前确认前端临时依赖目录没有残留，`node_modules` 和 `package-lock.json` 不会被带入仓库。

## 验证

- `npm run build` 通过。
- `go run build.go dev` 通过。
- `go test ./src/...` 通过。
- 本地 WebUI `http://127.0.0.1:8080` 启动验证通过。
