# v1.2.0 更新说明

本版本集中完善 WebUI 体验、外部工具管理器和 OpenList 云上传流程。相比 `v1.1.16`，这次不只是界面微调：云上传从配置展示推进到实际上传链路，设置页 GUI 补齐了更多 YAML 配置项，外部工具页也修复了加载失败、版本选择和深色主题可读性问题。

## OpenList 云上传

- 云上传阶段现在会真正调用 OpenList API 上传录制文件，不再只是记录日志或占位。
- 支持外部 OpenList：如果已经通过 Docker 或其他方式部署 OpenList，可以在配置中填写服务地址、令牌和存储位置，程序会优先使用外部实例。
- 保留内置 OpenList：未填写外部地址时，仍可由 RemoteTools 准备并使用内置 OpenList。
- 新增 OpenList 配置校验接口，便于在保存配置前确认地址、Token、存储位置是否能正常访问。
- 优化云上传路径模板，默认路径改为更清晰的录播归档目录结构，避免不同直播间文件混在一起。
- 设置页文案从“存储名称”调整为“存储位置”，更贴近 OpenList 实际使用语义。

## 设置页与 WebUI 操作体验

- GUI 模式补齐更多 YAML 配置项，减少必须切换到 YAML 模式才能配置的情况。
- 主页面右上角增加手动刷新和自动刷新控制，开启自动刷新后图标会持续旋转，状态更新不需要整页刷新。
- 自动刷新间隔调整得更短，适合观察录制状态、运行状态和外部组件准备状态。
- 主页面、设置页、弹幕页、文件页等长页面增加“回到顶部”浮标，滚动到底部后可以快速返回顶部。
- 录制可用后，辅助组件准备提示会进入 5 秒倒计时并自动隐藏，减少顶部提示长期占位。
- 登录页、主页面和外部页面继续统一 Codex 风格主题，并保持浅色、深色与多配色方案一致。

## 外部工具管理器

- 修复 `/tools/` 页面一直显示“加载工具失败”的问题：
  - RemoteTools 前端请求根路径 `/api/...` 时，现在会在 `/tools/` 页面自动改写到 `/tools/api/...`。
  - `/tools/api/status` 增加快速状态响应，避免单个工具下载状态扫描卡住整个工具列表首屏。
- 工具列表现在可以稳定显示所有工具组，包括 `ffmpeg`、`bililive-scheduler`、`biliLive-tools`、`openlist` 等。
- 多版本工具改为下拉选择默认版本，避免每个版本都展开成一张安装卡片。
- 修复展开详情时短暂闪过旧版“多安装卡片”布局的问题。
- 保留默认版本选择能力，切换后会保存为该工具组的默认版本。
- 深色主题下增强“未安装”“已安装”“已禁用”“运行平台”“默认版本”“版本号”等信息的区分度。
- 修复 Tools 页按钮、开关、空状态、工具组信息在深色主题下文字过亮、过暗或看不清的问题。

## 直播间列表与日志面板

- 修复直播间列表操作后运行状态不自动刷新的问题，状态会随轮询或手动刷新同步更新。
- 修复直播间展开详情在深色主题下背景、筛选栏、流列表和配置行不协调的问题。
- 修复最近日志面板仍然使用白底的问题，现在会跟随当前主题。
- 统一直播间列表、设置、日志、工具页中的按钮、卡片和开关样式，让深色主题下的对比更稳定。

## 主题与可读性

- 补齐 Codex 风格配色列表，并按字母顺序展示。
- 登录页主题配色顺序与登录后的主页面保持一致。
- 优化主题与配色下拉框的毛玻璃效果，减少原生下拉框的突兀感。
- 深色主题避免使用纯黑背景，降低覆盖原本黑色文字导致看不清的概率。
- Scheduler 和 Tools 外部页面会尽量继承主站主题变量，减少“白的太白、暗的太暗”的割裂感。
- iOS 风格开关启用后使用绿色状态，并优化按下时的滑块宽度和动画。
