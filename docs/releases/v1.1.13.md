# v1.1.13 更新说明

本版本重点修复 WebUI 默认公开访问带来的安全风险，建议所有通过 Docker、NAS、服务器或公网端口访问 WebUI 的用户升级。

## WebUI 安全

- 新增 `rpc.auth` 配置，支持为 WebUI、API、录制文件访问、Tools WebUI 和 Scheduler WebUI 统一开启 Basic Auth 登录保护。
- 开启鉴权后，访问页面或接口时需要输入用户名和密码，避免未授权用户直接修改配置、删除文件、触发更新或查看录制内容。
- 开启鉴权时会校验用户名和密码，用户名不能为空，密码也不能为空，避免误配置成空密码。
- 程序启动时如果检测到 WebUI 正在监听公开地址（例如 `:8080`、`0.0.0.0:8080`）且未开启 `rpc.auth`，会在日志中输出明确的安全提醒。

## 配置与文档

- `config.yml` 新增 `rpc.auth` 示例字段，并补充中文注释说明如何开启鉴权。
- README 中的 Docker 示例端口映射改为 `127.0.0.1:8080:8080`，默认只允许宿主机本机访问，降低误暴露到公网的风险。
- 文档补充说明：如果确实需要从局域网或公网访问 WebUI，请先开启 `rpc.auth` 并设置强密码，再按实际网络环境开放端口。

## 升级建议

- 如果你的 WebUI 只在本机访问，可以继续保持 `rpc.auth.enable: false`。
- 如果你的 WebUI 会被局域网、NAS 反向代理、云服务器公网 IP 或端口映射访问，建议立即开启：

```yaml
rpc:
  auth:
    enable: true
    username: admin
    password: "请改成你的强密码"
```

- Docker 用户如果不需要公网访问，建议把端口映射改为：

```bash
-p 127.0.0.1:8080:8080
```
