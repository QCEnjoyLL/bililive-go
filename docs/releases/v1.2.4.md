# v1.2.4 更新说明

本版本主要优化 WebUI 的外部工具管理体验，并补充若干主题、登录、配置编辑和更新检测修复。重点解决 Tools 页面多版本工具状态不一致、必需组件可误操作、深色主题显示不协调等问题。

## 外部工具管理

- 重构 Tools 页面多版本工具的展示方式，统一为“当前版本 / 可选版本 / 启用”流程，避免不同工具组出现旧版布局或留白。
- 修复展开工具详情时短暂闪现旧布局的问题，尤其是“锁定版本”等旧字段不再一闪而过。
- 修复多个版本共存时，切换启用版本后顶部“当前版本”和安装状态未及时刷新的问题。
- 未安装版本不再允许启用，启用按钮只会在“已安装且未启用”的版本上显示。
- 未安装版本不再显示可卸载操作，避免点击后出现“卸载完成”但实际没有任何变化的误导。
- 对 `ffmpeg`、`dotnet`、`node`、`biliLive-tools`、`bililive-recorder`、`bililive-scheduler` 等必需 / 默认安装工具增加保护：
  - 至少必须保留一个已安装版本，不能卸载所有版本。
  - 不允许停用必需工具组，录制和相关功能依赖这些工具保持启用。
  - 操作被拦截时会给出明确中文说明，而不是只显示笼统的“卸载失败”。
- 对 `bililive-go`、`klive` 等内部管理组件增加保护，禁止在外部工具页手动卸载、启用或禁用。
- 后端为工具卸载和启用接口增加兜底校验，避免绕过页面直接调用 API 导致必需工具进入不可用状态。
- 工具页新增工具分类说明，区分必需 / 默认安装、推荐安装、可选安装和内部管理组件。

## 主题与界面

- 继续优化 Tools、Scheduler 等外部页面的深色主题适配，减少白底过亮、暗块过暗、文字对比不足的问题。
- 统一 Tools 页面“已安装 / 未安装 / 已启用 / 未启用”等状态标签的字号、间距和视觉风格。
- 修复登录页输入框内容贴边的问题，增加合理内边距，并优化浏览器自动填充时的主题颜色。
- 修复登录页和外部页面部分文本在深色主题下颜色不跟随主题的问题。

## 配置页面

- YAML 模式编辑器改为主题适配更稳定的文本编辑区，修复深色主题下 YAML 内容几乎不可见的问题。
- 配置页保存按钮保持底部浮动，便于在长配置页面中随时保存。
- 优化云盘上传设置中的“上传时机”说明，明确“立即上传原始文件”和“处理流程结束后上传”的区别。

## 更新检测

- 修复本地 `-dev` 测试版检测远程同核心版本正式版时被误判为需要更新的问题。
- 例如本地 `v1.2.4-dev` 不会再把远程 `v1.2.4` 判断为更新；只有远程核心版本更高时才提示更新。
