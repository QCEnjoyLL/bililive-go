# v1.1.16 更新说明

本版本主要完善 WebUI 的主题与登录体验，并修复 `/tools/`、`/scheduler/` 外部工具页面在深色主题下文字和信息看不清的问题。同时继续补强录制诊断与网络统计，让后续定位 HLS 卡顿问题时能拿到更可靠的数据。

## WebUI 主题与登录体验

- 新增完整的 Codex 风格配色列表，包含 One、Absolutely、Ayu、Catppuccin、Codex、Dracula、Everforest、GitHub、Gruvbox、Linear、Lobster、Material、Matrix、Monokai、Night Owl、Nord、Notion、Oscurance、Raycast、Rose Pine、Sentry、Solarized、Temple、Tokyo Night、Vercel、VS Code Plus、Xcode。
- 主题支持浅色、深色和跟随系统，并保存在浏览器本地设置中；主 WebUI、登录页、外部工具页会尽量保持同一套外观。
- 登录页同步支持主题和配色切换，优化了下拉菜单的毛玻璃效果与长列表滚动体验。
- 顶部栏加入站点图标和更紧凑的布局，退出登录按钮可清理 WebUI 会话并返回登录页。

## 外部工具页显示修复

- 修复 `/scheduler/` 调度器页面深色主题下黑字、灰字、白底卡片没有正确适配的问题，提升仪表盘、任务说明、状态卡片的可读性。
- 修复 `/tools/` 外部工具管理器深色主题下工具名称、默认版本、安装状态、按钮等信息对比度不足的问题。
- 兼容常见 Ant Design、Tailwind、shadcn 风格 class，例如 `bg-white`、`bg-card`、`text-gray-900`、`text-muted-foreground` 等，减少外部页面局部样式漏改导致的深色主题不可读。
- 浅色主题下修复部分外部页面导航区域白底白字的问题。

## 录制与诊断改进

- 修复 HLS 录制相关网络统计口径，让录制中网络速度能更准确反映 MOUFLON HLS 分段下载流量。
- 继续保留并完善 HLS 诊断日志，用于区分下载失败、playlist 滑窗漏段、写入等待、源流视频帧本身停顿等不同原因。
- 优化部分 HLS 请求与统计代码，降低诊断信息与实际录制行为不一致的概率。
