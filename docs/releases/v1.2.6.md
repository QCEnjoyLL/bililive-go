# v1.2.6 更新说明

本版本修复开启 WebUI 登录保护后，调度器页面误判 BiliLive-go API 不可达的问题，并优化调度器统计信息在不同宽度下的显示效果。

## 调度器连接状态

- 修复启用 WebUI 鉴权后，Scheduler 后台健康检查访问主程序 API 被 `401` 拦截，导致页面显示“bililive-go API 不可达”的问题。
- Scheduler 现在会通过仅本机可用的内部 API 通道访问 BiliLive-go，不再依赖普通未登录 API 请求。
- 内部通道使用运行时随机 token，并限制为本机回环访问；不会把 WebUI 用户名或密码写入 Scheduler 启动参数，也不会放宽外部访问的登录保护。
- 普通 WebUI / API 访问仍保持原有鉴权逻辑，未登录访问受保护接口仍会被拦截。

## 调度器页面显示

- 修复调度器状态卡片中“运行中”等短状态文字在窄布局下被拆成多行的问题。
- 优化“运行时间”等较长统计值的布局：空间足够时保持单行显示，空间不足或运行时间变长时允许在标签和值之间换行，避免内容溢出或挤压。
- 继续保持 Scheduler 页面与主 WebUI 主题注入样式一致，减少外部页面在深色 / 浅色主题下的显示差异。
